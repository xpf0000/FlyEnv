## Overview

Consul 是 FlyEnv 集成的一个分布式服务网格解决方案，提供服务发现、服务配置和服务分段功能。在 FlyEnv 中，Consul 作为后台服务运行，通过 agent 模式启动，支持 Web UI 进行可视化管理。

相关文档链接:
- [Base Module](./base.md) - 继承的基类，包含通用服务生命周期管理
- [Nginx](./nginx.md) - 常与 Consul 配合使用的服务发现消费者
- [Service Manager](../components/ServiceManager/index.vue) - 通用服务管理组件

## Lifecycle Management

### Service Startup Flow

```
startService() [Base Class]
    │
    ├── Version validation
    ├── _linkVersion() [Homebrew]
    ├── _stopServer() [Terminate existing]
    └── _startServer() [Consul-specific]
            │
            ├── initConfig() - Create default JSON if missing
            ├── Prepare paths:
            │   ├── baseDir: {BaseDir}/consul
            │   ├── dbPath: {baseDir}/consul-{version}-data
            │   ├── logPath: {baseDir}/consul.log
            │   └── ip: getPrimaryLocalIPAddress()
            │
            └── Execute:
                Windows → serviceStartExecCMD()
                Linux/macOS → serviceStartExec(root=true)
```

### Start Command Construction

```bash
consul agent \
  -config-file="{iniFile}" \
  -data-dir="{dbPath}" \
  -log-file="{logPath}" \
  -bind="{ip}" \
  -pid-file="{pidPath}"
```

Sources: src/fork/module/Consul/index.ts:71-133

### Service Shutdown Flow

继承 Base 类的 `_stopServer()` 方法：

1. **PID Collection**:
   - 读取 `{BaseDir}/pid/consul.pid`
   - 搜索进程名匹配（注意：Consul 未在 `dis` 映射中定义特定进程名）

2. **Process Termination**:
   - Windows: `ProcessKill('-INT', pids)`
   - Linux/macOS: `ProcessKill('-INT', pids)` (default signal)

3. **Cleanup**:
   - 删除 PID 文件
   - 发送日志事件

Sources: src/fork/module/Base/index.ts:123-250

`initConfig()` 方法在首次启动时创建默认配置：

| Step | Action |

### Default Configuration

```json
{
  "server": true,
  "bootstrap": true,
  "client_addr": "127.0.0.1",
  "ui": true
}
```

**配置说明**:
- `server: true` - 以服务器模式运行（非客户端模式）
- `bootstrap: true` - 启用引导模式（适用于单节点开发环境）
- `client_addr` - HTTP API 和 UI 监听地址
- `ui: true` - 启用内置 Web UI

## Platform Differences

| Feature | macOS | Windows | Linux |
|---------|-------|---------|-------|
| Binary Name | `consul` | `consul.exe` | `consul` |
| Execution | `serviceStartExec(root: false)` | `serviceStartExecCMD()` | `serviceStartExec(root: true)` |
| Install Path | `{AppDir}/consul/{version}/consul` | `{AppDir}/consul/{version}/consul.exe` | `{AppDir}/consul/{version}/consul` |
| Package Format | `.zip` | `.zip` | `.zip` |
| Version Source | Homebrew, MacPorts, Manual | Manual | Manual |

**Notes**:
- Linux 需要 root 权限启动 (`root: true`)
- Windows 使用 `serviceStartExecCMD` 而非 `serviceStartExec`
- 所有平台使用相同的 ZIP 解压逻辑

## Troubleshooting

### Common Issues

| Issue | Possible Cause | Solution |
|-------|---------------|----------|
| Config not created | Directory permission | Check `{BaseDir}/consul` writable |
| Service won't start | Port conflict | Check 8300-8302, 8500 ports |
| Data dir not found | Custom path invalid | Reset via UI or clear localForage |
| Version not detected | Binary not executable | Check file permissions |

### Debug Commands

Check Consul status manually:
```bash
# Check version
consul version

# Validate config
consul validate /path/to/consul.json

# Check members
consul members

# View logs
tail -f ~/Library/Application\ Support/FlyEnv/consul/consul.log
```
