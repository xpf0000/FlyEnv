# FlyEnv Interface and Feature Overview Video Design

日期：2026-07-16

## 背景

源录屏 `docs/Kapture 2026-07-16 at 22.32.08.mp4` 展示 FlyEnv 的界面交互与主要功能，时长约 12 分 54 秒，分辨率为 3840×2160、30fps，且没有音轨。标准片头 `docs/flyenv-header.mp4` 时长约 5 秒，分辨率为 1920×1080，并包含片头音乐。

本次后期需要把标准片头接到录屏前，增加英文配音和英文字幕，让第一次接触 FlyEnv 的观众不仅知道画面中做了什么，也理解这些功能为什么这样设计，以及它们在真实开发工作中的价值。

## 目标

1. 完整保留原录屏中的界面操作和功能演示，不加速、不跳剪、不压缩流程。
2. 在开头加入 `docs/flyenv-header.mp4`，保留片头原有音乐。
3. 使用 Microsoft Edge Neural TTS 的 `en-US-BrianNeural` 生成稳重、自然的美式男声配音。
4. 生成与配音逐句一致的英文字幕，并同时交付烧录字幕版视频与独立 SRT 文件。
5. 让旁白覆盖“功能是什么、为什么这样设计、给用户带来什么好处”三个层次。
6. 输出适合 YouTube、Bilibili 和项目文档直接使用的 1920×1080 成片。

## 非目标

- 不把视频剪成短视频或宣传片。
- 不删除正常的鼠标移动、窗口切换或完整操作步骤。
- 不加入额外背景音乐；片头结束后以清晰的英文旁白为主。
- 不重新设计片头，也不增加新的片尾动画。
- 不在旁白中逐项朗读所有版本号、按钮文字或模块名称。
- 不声称录屏中没有明确展示的能力。

## 成片结构

### 1. 标准片头

- 直接使用 `docs/flyenv-header.mp4`。
- 保留片头音乐和原始动画节奏。
- 片头结束时将音乐自然淡出，随后进入 FlyEnv 录屏。
- 旁白不覆盖片头最密集的音乐段；第一句介绍在片头末尾或录屏开始后进入。

### 2. 完整功能总览

录屏全程保持 1.0x，按画面实际顺序组织旁白。旁白内容归纳为以下主题：

1. FlyEnv 的整体界面与模块化结构。
2. 模块的显示和隐藏。
3. 模块版本的浏览与下载。
4. 服务状态管理和一键启动能力。
5. 面向不同项目的自定义启动组。
6. 本地站点及相关服务的管理体验。
7. 内置开发工具，包括编码、截图和图片处理等功能。
8. 多语言、主题和其他个性化设置。

主题数量比用户列出的七个功能块多一个，是因为开场需要先建立对 FlyEnv 整体定位和界面结构的认识；其余内容仍完整覆盖用户指定的七个重点。

### 3. 自然收尾

- 保留原录屏末尾仍在 FlyEnv 内置工具和启动组界面中的有效画面。
- 最后一段旁白总结 FlyEnv 的核心价值：按需组合模块、统一管理本地服务，并让同一个工具适应不同技术栈和项目。
- 旁白结束后保留短暂停顿，让观众看清最终界面，然后淡出音频并结束视频。

## 旁白设计

### 声音

- 引擎：Microsoft Edge Neural TTS，通过 `edge-tts` 生成。
- Voice ID：`en-US-BrianNeural`。
- 语言：美式英语。
- 基础语速：`-6%`。
- 表达方式：稳重、友好、清楚，接近产品导览而不是广告播音。

### 内容深度

每个主要功能段落使用三层表达：

1. **What**：说明当前画面正在演示什么。
2. **Why**：解释 FlyEnv 为什么采用这种产品设计。
3. **Benefit**：说明这种设计如何降低配置成本、认知负担或项目切换成本。

例如，模块显示与隐藏不能只说用户可以切换模块，还需要解释：FlyEnv 支持的技术栈很多，但单个开发者通常只需要其中一部分；按需显示可以让界面保持专注，同时又不牺牲以后启用其他技术栈的能力。

启动组部分需要解释：不同项目依赖的 Web Server、语言运行时和数据库组合不同；保存启动组能够把一组服务变成可重复的一键工作流，减少每次切换项目时的手工操作。

### 节奏

- 旁白以主题段落为单位，不要求从头到尾持续说话。
- 重要界面切换后先留出短暂观察时间，再开始解释。
- 在列表滚动、版本浏览或工具实际操作期间，允许用较完整的句子展开设计原因和使用收益。
- 避免旁白提前描述尚未出现在画面中的功能。
- 相邻主题之间保留自然停顿，避免让十几分钟的配音听起来像连续朗读。

## 字幕设计

- 字幕文本与实际配音逐句一致，不另写缩略版字幕。
- 同时输出独立 SRT 和烧录字幕版 MP4。
- 每条字幕优先保持一到两行，按照自然语义断句。
- 字幕位于画面下方安全区域，避开 FlyEnv 底部按钮和系统 Dock。
- 使用高对比度白色文字、深色描边或半透明背景，保证浅色和深色主题下都清晰可读。
- 字幕时间以生成后的真实 TTS 音频长度为准，不以估算阅读速度强行切分。

## 视频与音频处理

### 视频

- 原录屏从头到尾按 1.0x 保留。
- 将 4K 录屏等比缩放到 1920×1080。
- 片头统一转换为 30fps，确保与主视频一致。
- 输出编码为 H.264，并启用 `+faststart`。
- 不改变原录屏的画面比例，不裁掉 FlyEnv 界面或浏览器区域。

### 音频

- 保留片头自带 AAC 音频。
- 主录屏没有原始音频，因此录屏部分只使用英文旁白，不伪造环境声或点击声。
- 片头音乐结束和第一句旁白之间使用平滑的音量过渡。
- TTS 分段生成后合成为一条完整配音轨，便于单独调整配音而无需重新编码视频。
- 最终音频使用 AAC，并控制峰值避免片头音乐明显响于旁白。

## 验证标准

1. 最终视频以完整标准片头开场，片头音乐可正常播放。
2. 原录屏的全部有效画面和操作都保留，播放速度为 1.0x。
3. 英文配音使用 `en-US-BrianNeural`，没有句子被截断、重叠或明显抢拍。
4. 旁白完整解释模块开关、版本下载、服务管理、启动组、站点、内置工具、多语言和主题设置。
5. 每个主要功能至少包含一处设计原因或用户收益说明。
6. 烧录字幕与旁白内容、开始时间和结束时间一致。
7. 独立 SRT 可被播放器正常识别。
8. 最终文件为 1920×1080、30fps、H.264 视频和 AAC 音频，并包含 faststart 元数据。
9. 抽查片头、三个中段主题和结尾画面，字幕不遮挡关键控件，画面无黑帧或异常跳转。

## 风险控制

- 如果 `edge-tts` 受系统 Python 环境限制，在任务目录创建独立 `.venv`，不修改项目全局 Python 环境。
- TTS 按段生成并保留中间文件；修改某句旁白时只重建配音与最终封装，不重新编码完整基础视频。
- 渲染前先用低分辨率或短片段验证字幕样式和音量，再执行完整 1080p 编码。
- 保持源录屏与标准片头不变，所有输出写入新的任务目录。
