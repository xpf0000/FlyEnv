# FlyEnv 启动组功能方案

关联需求：

- #757：新增总控入口、启动组卡片和多组管理。
- #742：允许语言项目单独决定是否参与启动组。

## 一、侧边栏结构

在侧边栏所有分类最上方新增“控制台”分类，其下新增“启动组”模块。

控制台
└── 启动组

Web 服务
├── Nginx
├── Apache
└── ...

数据库
├── MySQL
└── ...

启动组模块纳入现有“设置 → 模块显示”配置：

- 用户可以显示或隐藏启动组模块。
- 控制台下没有可见模块时，整个控制台分类自动隐藏。
- 隐藏模块不会删除启动组及默认组配置。
- 重新显示后恢复原来的启动组配置。

## 二、启动组页面

页面以卡片形式显示所有启动组。

每张卡片包括：

- 启动组名称。
- 服务数量和运行数量。
- 当前状态。
- 成员概要。
- 启动、停止、编辑、删除按钮。
- 卡片尾部的“默认启动组”开关。

卡片状态包括：

- 已停止：成员全部停止。
- 运行中：成员全部运行。
- 部分运行：只有部分成员运行。
- 执行中：正在启动或停止。
- 配置异常：存在已删除的版本或项目。

页面提供“新增启动组”按钮。没有启动组时显示功能说明和空状态。

## 三、新增与编辑启动组

新增启动组分为三个步骤。

### 1. 基础信息

用户填写：

- 启动组名称。
- 可选描述。
- 可选颜色或图标。

### 2. 选择服务

展示当前系统中已安装、当前平台支持并且能够独立启停的服务。

#### 服务模块具体版本

允许选择某个服务模块的具体安装版本，例如：

Nginx
├── 1.26.3
└── 1.27.4

MySQL
├── 8.0.42
└── 8.4.5

PHP-FPM
├── PHP 8.2.28
├── PHP 8.3.22
└── PHP 8.4.8

规则：

- 选择的是具体版本，不是模块当前选中的版本。
- 启动组执行时不能自动替换为模块的当前版本。
- 允许选择同一模块的多个版本。
- 存在端口冲突风险时显示提示，但不强制禁止。
- 模块是否在侧边栏显示，不影响其服务版本出现在选择列表中。

#### 语言模块自定义项目

列出各语言模块中能够作为服务运行的自定义项目，例如：

Node.js
├── API Server
└── Admin Dev Server

Python
└── Data Service

Go
└── Gateway

规则：

- 只显示支持独立启停的项目服务。
- 项目保留在项目列表中，但可以不加入任何启动组。
- 项目需要使用稳定 ID 引用，不能只依赖项目路径。
- 项目路径变化或项目被删除后，启动组成员显示为失效。

第一期不将普通 Apache/Nginx 虚拟主机作为独立项目服务处理，因为虚拟主机没有独立进程。

### 3. 设置启动顺序

用户通过拖动调整服务启动顺序，例如：

1. MySQL 8.4
2. Redis 7
3. PHP-FPM 8.3
4. Node.js API Server
5. Node.js Admin Dev Server

这样可以确保数据库、缓存和后端服务先于依赖它们的项目启动。

## 四、启动与停止规则

### 启动启动组

- 按用户设置的顺序逐个启动。
- 已经运行的成员直接跳过。
- 某个成员启动失败时，停止启动后续成员。
- 保留已经成功启动的成员，不自动回滚。
- 最后显示成功、跳过、失败和未执行的成员。

### 停止启动组

- 按启动顺序的倒序逐个停止。
- 已经停止的成员直接跳过。
- 某个成员停止失败时，继续停止其他成员。
- 最后统一显示所有失败信息。

同一服务可以存在于多个启动组中。停止某个启动组时，直接停止该组中的服务，不判断服务是否由其他启动组或用户手动启动。

## 五、默认启动组

系统只允许设置一个默认启动组。

数据层建议保存：

defaultStartupGroupId?: string

不在每张卡片中分别保存 isDefault，避免出现多个默认组。

卡片开关行为：

- 开启某张卡片的默认开关时，原默认组自动取消。
- 关闭当前默认组开关后，系统进入无默认组状态。
- 空启动组不能设置为默认组。
- 删除默认启动组时自动清空默认配置。
- 隐藏启动组模块时保留默认组配置，但默认组暂时不生效。
- 重新显示启动组模块后，原默认组恢复生效。

## 六、现有群组按钮行为

侧边栏现有群组启动/关闭按钮按下面的规则执行：

启动组模块    默认启动组            执行逻辑
━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━━━━  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━
显示          存在有效默认组        操作默认启动组
────────────  ────────────────────  ────────────────────────────
显示          没有默认组            使用现有侧边栏逻辑
────────────  ────────────────────  ────────────────────────────
隐藏          无论是否存在默认组    使用现有侧边栏逻辑
────────────  ────────────────────  ────────────────────────────
显示          默认组已被删除        清除失效配置并使用现有逻辑

“现有侧边栏逻辑”保持不变：

- 根据侧边栏显示的模块进行群组启停。
- 继续使用当前版本。
- 保留当前模块参与群组启动的配置和判断。

如果默认组存在，但部分成员已经失效：

- 执行默认组中其余有效成员。
- 将失效成员显示为配置错误。
- 不允许回退并启动侧边栏中的全部模块，避免意外启动额外服务。

## 七、隐藏启动组模块

隐藏启动组模块相当于停用启动组功能，因此必须先停止启动组包含的服务。

执行流程：

1. 用户在模块设置中关闭启动组模块。
2. 系统提示隐藏操作将停止所有启动组中的服务。
3. 用户确认后，收集所有启动组的全部成员。
4. 按各启动组的倒序依次生成停止队列。
5. 对重复服务去重，每个服务只停止一次。
6. 无论服务由启动组还是用户手动启动，只要属于任意启动组，就直接停止。
7. 单个成员停止失败时继续处理其他成员。
8. 全部停止成功后隐藏启动组模块。
9. 存在停止失败时保留启动组模块，并显示失败成员。
10. 启动组正在执行启停操作时，禁止修改其显示状态。

隐藏后：

- 启动组及默认组配置继续保留。
- 侧边栏群组按钮恢复现有逻辑。
- 重新显示模块不会自动启动任何服务。
- 重新显示后，原默认启动组重新成为群组按钮的执行目标。

## 八、数据模型

interface StartupGroup {
id: string
name: string
description?: string
color?: string
items: StartupGroupItem[]
createdAt: number
updatedAt: number
}

type StartupGroupItem =
| {
id: string
type: 'service-version'
module: AppModuleEnum
versionBin: string
}
| {
id: string
type: 'language-project'
module: AppModuleEnum
projectId: string
}

interface StartupGroupConfig {
groups: StartupGroup[]
defaultStartupGroupId?: string
}

成员数组顺序就是启动顺序，不需要额外保存 order。

服务版本以模块类型和安装路径共同定位。自定义项目使用稳定的项目 ID 定位。

## 九、执行层设计

启动组页面不应直接调用侧边栏组件中的 groupDo()，建议增加独立执行层：

StartupGroupRunner
├── ServiceVersionAdapter
├── PhpFpmAdapter
└── LanguageProjectAdapter

适配器统一提供：

interface StartupGroupAdapter {
getState(item: StartupGroupItem): Promise<ServiceState>
start(item: StartupGroupItem): Promise<void>
stop(item: StartupGroupItem): Promise<void>
exists(item: StartupGroupItem): Promise<boolean>
}

现有侧边栏逻辑继续保留。群组按钮只负责根据模块可见性和默认组配置，决定调用启动组执行器还是原有 groupDo()。

## 十、实施阶段

### 第一阶段：基础模型与执行器

- 增加启动组数据结构和持久化。
- 增加具体服务版本执行适配器。
- 增加 PHP-FPM 执行适配器。
- 增加语言项目执行适配器。
- 实现顺序启动和逆序停止。

### 第二阶段：控制台和启动组页面

- 增加控制台分类。
- 实现服务选择和拖动排序。
- 接入模块显示设置。

### 第三阶段：默认启动组

- 实现唯一默认组。
- 接入现有群组启停按钮。
- 实现没有默认组时的兼容回退。
- 实现部分运行和配置异常状态。

### 第四阶段：隐藏处理

- 隐藏前停止所有启动组成员。
- 实现重复成员去重。
- 实现停止失败时阻止隐藏。
- 隐藏后恢复现有侧边栏群组逻辑。

## 十一、验收标准

- 控制台位于所有侧边栏分类最上方。
- 用户可以隐藏启动组模块和控制台分类。
- 可以创建多个启动组。
- 可以选择服务模块的具体安装版本。
- 可以选择多个 PHP-FPM 版本。
- 可以选择各语言模块的自定义项目服务。
- 可以拖动设置启动顺序。
- 启动按正序执行，停止按倒序执行。
- 系统最多只有一个默认启动组。
- 默认组存在时，现有群组按钮只操作默认组。
- 启动组隐藏或没有默认组时，群组按钮保持原有逻辑。
- 隐藏启动组前会停止所有启动组包含的服务。
- 手动启动但属于启动组的服务同样会被停止。
- 服务版本或项目失效时不会自动替换或启动其他服务。
