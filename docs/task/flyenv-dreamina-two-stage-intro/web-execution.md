# 即梦网页版执行说明

这套素材用于生成两段视频。第一段负责模块爆发、球体内部运镜和坍缩；第二段负责三个开关、闪电和精确的 `FlyEnv` 落版。不要把六张图片一次性全部交给同一个任务。

## 第一段：模块与球内运镜

在即梦网页版选择支持多图参考的视频生成模式，按以下固定顺序上传：

1. 图片 1：`references/module-board-a.png`
2. 图片 2：`references/module-board-b.png`
3. 图片 3：`references/module-board-c.png`
4. 图片 4：`references/logo-off.png`
5. 图片 5：`references/interior-camera.png`

确认网页中的 `@图片1` 至 `@图片5` 与上述顺序一致，然后粘贴 `clip-a-prompt-zh.txt`。

网页参数：

- 时长：5 秒
- 画幅：必须在网页参数中选择 `16:9`
- 清晰度：选择账号当前可用的最高档
- 分镜：关闭智能分镜或多镜头
- 声音：若界面提供原生音频选项则开启

不要只在提示词里写 16:9；之前的三个文件虽然提示词写了 16:9，实际输出仍然是 960×960。

第一段不要上传 `end-card.png`，否则准确的 `FlyEnv` 字标可能提前泄漏进模块镜头。

生成后先检查：输出是否真的是 16:9；是否从显示器屏幕进入球心并向外看；是否出现完整外部球体；所有可读名称是否正确；最后是否收成居中 Logo 与青白闪光。第一段不合格时不要急着生成第二段。

## 第二段：开关与最终落版

在即梦网页版选择“首尾帧生视频”模式：

- 首帧：`references/logo-off.png`
- 尾帧：`references/end-card.png`
- 提示词：`clip-b-prompt-zh.txt`

网页参数：

- 时长：4 秒
- 画幅：由两张 1920×1080 首尾帧确定，但仍需确认界面显示 `16:9`
- 清晰度：选择账号当前可用的最高档
- 分镜：关闭智能分镜或多镜头
- 声音：若界面提供原生音频选项则开启

生成后逐帧检查：顶部、中间、底部开关是否依次滑动；每次是否只有一个轨道变绿；已开启开关是否保持；闪电是否完整绕一圈；最终是否严格停在尾帧的 `FlyEnv`，而不是重新生成 `FlyEnb`。

## 后续合成

把两段原始 MP4 保持原样交回当前工作区即可。推荐让第一段最后 1 秒和第二段最初 1 秒都保持居中 Logo 与青白闪光，再用 1 秒视频/音频交叉淡化合成：`5 + 4 - 1 = 8 秒`。
